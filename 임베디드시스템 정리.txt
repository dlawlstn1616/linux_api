임베디드 : 무언가를 단단히 고정시키다의 수동형

임베디드 시스템 : 다른 시스템에 컴퓨팅 시스템을 얹은 것
			PC와는 다르게 특정 기능을 위한 전용 컴퓨터 시스템
			특정 기능을 위해 HW와 SW를 밀접하게 결합시키는 컴퓨팅 장치

컴퓨팅 시스템 = 마이크로프로세서 + 메모리 + 입출력 장치 + 소프트웨어
(임베디드 H/W : 프로세서, 메모리, 입출력 장치, 네트워크)
(임베디드 S/W : OS, 미들웨어(라이브러리, JVM이나 네트워크 프로토콜들), 어플리케이션)

특정 기능 = 일반적인 목적이 아니라 특정 목적을 위해 사용
-> 반드시 고성능 프로세서나 큰 메모리가 필요로 하지 않는다. -> 해당 기능만을 위한 성능이면 충분

=> 프로그램 사이즈는 최대한 적은 메모리를 가지고, 효율적인 알고리즘을 통해 저전력 CPU로 동작해야되며
     대기나 절전 모드를 통해 전력을 세이브하고, 신뢰도가 높아야 한다.

쉘 : 하드웨어 인터프리터 명령에 독립적
- 유저와 상호작용 하는 기능
- 입력을 읽고 명령을 실행(인터프리터)
- DOS의 command.com과 유사
- bash, csh, sh 등등

파일 타입으론
- 일반 파일 : 텍스트나 바이너리 파일
- 경로
- 장치 파일 : 하드웨어 장치를 표현
- 링크 파일
 * 하드 링크 : 경로로 파일에 접근, 같은 파일 시스템에서만 가능
 * 심볼릭 링크 : 파일에 경로를 저장

이름은 최대 255, 대소문자 구별
절대경로, 상대경로


커널 : 하드웨어에 의존적, 메모리 관리, 프로세스 스케쥴링, 입출력, 파일 메니지먼트
-> OS의 메모리 거주 지역
- 프로세스 관리 : 프로세스 생성 및 파괴, 스케쥴링, 동기화
- 메모리 관리 : 가상 메모리 관리
- 파일 시스템 : 여러 파일 시스템 지원
- 장치 관리 : 입출력 요청 스케쥴링, 메모리와 장치 간에 데이터 전송
- 네트워크 관리 : 통신 프로토콜

어플레케이션들은 시스템 콜이라는 커널 시스템을 사용한다

커널 버전 x.y.z
x : 주요 버전 넘버, 중요한 변화
y : 서브 버전 넘버, 짝수면 안정 홀수면 개발중
z : 릴리즈 넘버, 기능 추가나 버그 수정과 같은 작은 변화
uname -r
